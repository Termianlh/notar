The problem

A valid-looking message proves nothing. A signature proves everything.

As agents start acting for us — booking, negotiating, paying other people's agents — one question decides everything: is this agent who it claims to be?

Proof

Watch an impostor get rejected.

python examples/demo.py
① Alice creates an identity did:key:z6Mkg8bf168b99TWdxF16FkfmKeCRsiAJTzqYrHNTZsFBSHx ② Alice signs a card and registers it registered ③ Bob looks up Alice and verifies her Alice is who she claims to be (status: active) ④ Eve forges a card claiming to be Alice rejected — signature doesn't match Alice's key (422) ⑤ A tampered card is checked invalid — one changed byte breaks the signature ⑥ Eve registers as herself registered # being yourself is fine — impersonating isn't

Eve can claim Alice's identity, but she can't sign like Alice — because she doesn't hold Alice's private key. The registry refuses to store any card whose signature doesn't match the identity it claims.

Quickstart

Four lines to a verifiable identity.

from notar import create_identity, sign_card, verify_card # give your agent a verifiable identity ident = create_identity() print(ident.did) → did:key:z6Mk… # sign an agent card — its "business card" card = sign_card({ "issuer": ident.did, "name": "Alice", "endpoint": "https://alice.example/a2a", }, ident.private_key) # anyone can verify it — offline, no server verify_card(card) → True card["name"] = "evil-alice" verify_card(card) → False

install  →  git clone https://github.com/Termianlh/notar && cd notar && pip install -e .

How it works

Standard cryptography, put to work.

IdentityEd25519 & did:key
Each agent holds a keypair. Its public key is encoded directly into its DID, so the identifier is self-certifying — no lookup needed to check a signature.
Tamper-proofSigned agent cards
A card carries identity, endpoint, and capabilities, signed as one object. Change a single field and verification fails — the contact details are protected exactly like the identity.
No server neededVerifiable offline
Checking a signature is pure math. If you hold someone's card, you can verify it on your own machine. The private key never leaves its owner.
Trust, not trafficRegistry on the trust path
The registry handles discovery, standing, and — soon — reputation. Agents find and verify each other through it, but message each other directly.
